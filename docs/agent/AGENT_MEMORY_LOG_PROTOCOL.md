# Agent 跨窗口记忆记录规范（强制）

## 1. 目标
- 确保任意窗口中断后，都能用一份 Markdown 继续任务。
- 每一步都留下“已完成事项 + 证据 + 下一步”，避免记忆丢失。

## 2. 记录文件与位置
- 主项目目录固定为：`/home/kt/ai/luma-architecture`
- 主记录文件固定为：`/home/kt/ai/luma-architecture/docs/agent/AGENT_WORKLOG.md`
- 所有 agent 在每个步骤结束后，必须更新该文件。
- 不允许只在对话里汇报、不落盘。

## 3. 何时必须更新
- 完成任何一个可独立描述的动作后必须更新一次（最小粒度：一步）。
- 发生失败、回滚、阻塞时必须更新一次。
- 在切换窗口、切换 agent、切换阶段前必须更新一次。

## 4. 每次更新的最小字段（必填）
- 时间（本地时区）
- 当前阶段/子任务
- 本步目标
- 已完成事项（可核验）
- 产出物路径（文件/脚本/配置）
- 运行命令与关键输出摘要
- 遇到的问题与处理
- 下一步（单一、明确、可执行）

## 5. 记录格式模板（必须使用）

```md
## [YYYY-MM-DD HH:mm] Step N - <标题>
- 阶段: <例如 阶段0C Mamba3算子构建>
- 本步目标: <一句话>
- 已完成:
  - <完成事项1>
  - <完成事项2>
- 产出物:
  - <绝对路径1>
  - <绝对路径2>
- 执行命令:
  - `<命令1>`
  - `<命令2>`
- 关键结果:
  - <结果摘要1>
  - <结果摘要2>
- 问题/风险:
  - <问题或无>
- 下一步:
  - <下一步唯一动作>
```

## 6. 质量要求
- 每条“已完成”必须可追溯到文件或命令输出。
- 不写“基本完成/差不多”这类模糊描述。
- 下一步只能写一个优先动作，避免歧义。
- **每份实验报告必须包含至少一张 Markdown 数据表**（例如 candidate/stage/score/guard/key-metrics）。
- 报告中的结论段必须引用该表，不允许只有文字结论没有表格证据。

## 7. 交接要求
- 交接前必须补一条“交接快照”：
  - 当前整体状态
  - 最近一次成功验证
  - 未完成阻塞点
  - 接手者第一步该做什么

## 8. 与主规划联动
- 本规范默认绑定：
  - `/home/kt/ai/luma-architecture/docs/plans/Luma_v0.7.2_Agent_MasterPlan.md`
- `AGENT_WORKLOG.md` 中每条记录必须标注对应主规划阶段（如 `阶段0D`、`Gate P3`）。
