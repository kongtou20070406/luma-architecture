# Agent 跨窗口记忆记录规范（强制，含 Mechanical Overrides）

## 1. 目标
- 确保任意窗口中断后，都能用一份 Markdown 继续任务。
- 每一步都留下“已完成事项 + 证据 + 下一步”，避免记忆丢失。
- 让后续接手者可在 10 分钟内定位当前状态、边界和下一动作。

## 2. 记录文件与位置
- 主项目目录固定为：`/home/kt/ai/luma-architecture`
- 主记录文件固定为：`/home/kt/ai/luma-architecture/docs/agent/AGENT_WORKLOG.md`
- 所有 agent 在每个步骤结束后，必须更新该文件。
- 不允许只在对话里汇报、不落盘。

## 3. 何时必须更新
- 完成任何一个可独立描述的动作后必须更新一次（最小粒度：一步）。
- 发生失败、回滚、阻塞时必须更新一次。
- 在切换窗口、切换 agent、切换阶段前必须更新一次。
- 完成“目录清理/文件删除/路径迁移”后必须更新一次（避免后续找不到工件）。

## 4. 每次更新的最小字段（必填）
- 时间（本地时区）
- 当前阶段/子任务
- 本步目标
- 已完成事项（可核验）
- 产出物路径（文件/脚本/配置）
- 运行命令与关键输出摘要
- 遇到的问题与处理
- 有效性边界（哪些结果可用、哪些不可用于结论）
- 下一步（单一、明确、可执行）

## 5. 记录格式模板（必须使用）

```md
## [YYYY-MM-DD HH:mm] Step N - <标题>
- 阶段: <例如 阶段0C Mamba3算子构建>
- 本步目标: <一句话>
- 已完成:
  - <完成事项1>
  - <完成事项2>
- 产出物:
  - <绝对路径1>
  - <绝对路径2>
- 执行命令:
  - `<命令1>`
  - `<命令2>`
- 关键结果:
  - <结果摘要1>
  - <结果摘要2>
- 有效性边界:
  - <有效证据>
  - <无效证据与原因>
- 问题/风险:
  - <问题或无>
- 下一步:
  - <下一步唯一动作>
```

## 6. 质量要求
- 每条“已完成”必须可追溯到文件或命令输出。
- 不写“基本完成/差不多”这类模糊描述。
- 下一步只能写一个优先动作，避免歧义。
- 每份实验报告必须包含至少一张 Markdown 数据表（例如 `candidate/stage/score/guard/key-metrics`）。
- 报告结论段必须引用该表，不允许只有文字结论没有表格证据。

## 7. Mechanical Overrides（强制执行）

### 7.1 STEP 0 Rule（大文件重构前先清死代码）
- 对于单文件超过 `300 LOC` 且要做结构改造时：
- 必须先清理 dead code（unused props/exports/imports/debug logs）。
- 该清理必须以独立阶段记录（推荐独立 commit）。

### 7.2 分阶段执行（PHASED EXECUTION）
- 不允许一次性跨很多文件大改。
- 每个 phase 最多触达 `5` 个文件。
- Phase 1 完成并验证后，等待明确确认再进入 Phase 2。

### 7.3 Senior Dev Override
- 遇到架构不一致、状态重复、模式混乱时，不允许只做“最小修补”。
- 必须提出并落地结构化修复方案（并写明风险与收益）。

### 7.4 强制验证（FORCED VERIFICATION）
- 不得把“文件写入成功”当作“任务成功”。
- JS/TS 项目必须执行：
  - `npx tsc --noEmit`
  - `npx eslint . --quiet`（若配置）
- 非 JS/TS 项目必须执行该项目等价校验（例如 `py_compile`、测试脚本）。
- 若项目无类型检查/静态检查配置，必须在记录中明确写“无配置”，不得伪称已验证。

### 7.5 子代理并行（SUB-AGENT SWARMING）
- 当任务触及 `>5` 个相互独立文件时，默认采用并行子任务（5~8 文件/组）。
- 每组独立上下文，最后主代理统一整合。

### 7.6 上下文衰减防护（CONTEXT DECAY AWARENESS）
- 对话轮数较多（`10+`）后，编辑前必须重读目标文件。
- 不允许仅凭记忆编辑。

### 7.7 文件读取预算（FILE READ BUDGET）
- 单次读取最多按 `2000` 行窗口处理。
- 大文件（`>500 LOC`）必须分块顺序读取，不允许假设一次读取完整。

### 7.8 工具结果截断意识（TOOL RESULT BLINDNESS）
- 若结果异常短或可疑，必须重跑更窄范围命令（单目录/更严 glob）。
- 日志中需明确标注“疑似截断，已重跑”。

### 7.9 编辑完整性（EDIT INTEGRITY）
- 每次编辑前必须重读文件，编辑后再次读取确认改动已生效。
- 同一文件连续编辑不超过 3 次；超过时必须插入一次验证读。

### 7.10 无 AST 语义搜索时的重命名安全
- 任何符号改名都必须分开检查：
- 直接调用与引用
- 类型层引用（接口、泛型）
- 字符串字面量
- 动态导入 / `require()`
- re-export / barrel
- 测试与 mock

## 8. 交接要求
- 交接前必须补一条“交接快照”：
  - 当前整体状态
  - 最近一次成功验证
  - 未完成阻塞点
  - 接手者第一步该做什么
