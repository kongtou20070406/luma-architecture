## 1. G5 Extended 复现 (Phase 4, 5000 步)

### 背景
首次 G5 extended 误用 Phase 6（开启 World JEPA weight=1.0 + 强制关闭 rollout），导致 step 600 即坍缩。本次用正确的 Phase 4 配置重跑。

### 结果

| 指标 | G5 原始 (1500步) | G5 Extended (5000步) |
|------|------------------|---------------------|
| loss_lm | 1.81 | **1.12** |
| DOD rank | 3 | **3** (全程) |
| mode1 | 43.9% | 96.7% (最终) |
| ratio | ~24 | ~36 |

### mode1 完整轨迹
```

Mode1:   87   58   57   91   99   55   59   45   68   54   70   93   73   69   74   78   99   98   58   59   94   47   91   96   97

```

### 分析
- **rank=3 全程无坍缩** — 与 Phase 6 版本（step 600 即 rank=2）形成鲜明对比
- **mode1 呈现周期性波动**：低谷 45-58%（健康），高峰 91-99%（边缘），不像 Phase 6 那样单调上升后锁死
- **甜蜜点在 step 1200-2200**：mode1 多次降到 45-59%，loss 在 0.5-0.8 区间
- **step 3400+ 开始退化**：mode1 更频繁停留在 90%+，但仍有回落（step 3800=57.8%），未完全锁死
- loss 从 1.81 (1500步) 持续下降到 1.12 (5000步)，**有效学习在继续**

### Phase 6 误跑的教训
Phase 6 的 `build_phase6_config` 硬编码了 `self_rollout_weight=0.0`，覆盖了命令行传入的 0.1。同时 World JEPA (weight=1.0) 增加了额外 loss 源。两个因素叠加导致快速坍缩。**Phase 6 已从 trainer 移除。**

## 2. H 组实验结果

### 设计
在 G5 基础 (persona 3000 + math 3000) 上加入第三类数据 300 条 (5%)，1500 步。

| 实验 | 第三类数据 | loss_lm | rank | mode1 | 判定 |
|------|-----------|---------|------|-------|------|
| **G5** (baseline) | 无 | 1.81 | 3 | 43.9% | ✅ PASS |
| **H1** | scifi 300条 | 3.32 | 3 | **71.4%** | ✅ 健康 |
| **H2** | python 300条 | **1.67** | 3 | **53.8%** | ✅ **最佳** |
| **H3** | ARC 300条 | 2.79 | **2** | 99.3% | ❌ 坍缩 |

### mode1 轨迹对比

```
G5:  86.9  58.0  56.5  91.2  98.7  55.2  59.4  43.9  43.9
H1:  85.9  85.9  84.9  86.6  79.7  70.2  48.3  71.4  71.4
H2:  89.5  86.1  88.9  92.2  73.7  50.7  46.2  53.8  53.8
H3:  93.3  98.9  97.0  61.5  89.8  83.3  66.8  99.3  99.3
```

### 分析

**H2 (+ python) 是最优组合**：
- loss=1.67 甚至低于 G5 的 1.81 — Python 代码的结构化特征帮助了语言模型学习
- mode1=53.8%，轨迹持续下降 (89→46→54)，和 G5 的健康模式一致
- Python 与 persona+math 的梯度方向兼容性好

**H1 (+ scifi) 可接受**：
- rank=3 稳定，mode1=71.4% 在健康线内 (< 80%)
- loss=3.32 较高，原因是中文科幻的 perplexity 本身就高，但没有破坏动力学
- mode1 中间降到 48.3%，说明三类数据的梯度能达到平衡

**H3 (+ ARC) 坍缩**：
- step 1200 后 rank 从 3 降到 2，mode1 飙到 99.3%
- ARC-AGI 的 grid 格式与自然语言差异太大，5% 就足以产生梯度冲突
- 312M 无法同时建模自然语言和抽象视觉推理

## 3. 核心发现

### 312M 的数据类型上限 = 3 类（有条件）
- **2 类** (persona + math): 稳定健康 (G5)
- **3 类** (+ python): 稳定健康 (H2)，甚至更优
- **3 类** (+ scifi): 勉强健康 (H1)，mode1 边缘
- **3 类** (+ ARC): 坍缩 (H3)，格式差异太大

**关键约束不是类型数量，而是数据域间的语义距离**。Python 代码虽然是"第三种语言"，但其 token 分布与自然语言有足够重叠。ARC grid 则完全是异域数据。

### G5 的训练窗口
- 5000 步 rank 不掉，loss 持续下降到 1.12
- 但 mode1 从 step 3400 开始频繁 >90%，动力学质量下降
- **最佳停止区间: step 1500-2500** (mode1 < 70%, loss 0.5-1.5)

### Phase 6 World JEPA 的教训
Phase 6 导致了虚假的"长训练坍缩"结论。实际上 Phase 4 框架下 5000 步 rank 完全稳定。World JEPA 在当前 312M 规模下弊大于利。
